# 1. Register/Login first
curl -X POST http://localhost:3001/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"user@example.com","password":"password123"}'

# Save the access_token from response

# 2. Get Profile
curl -X GET http://localhost:3001/users/profile \
  -H "Authorization: Bearer YOUR_TOKEN"

# 3. Update Profile
curl -X PATCH http://localhost:3001/users/profile \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"New Name"}'

# 4. Create Preferences
curl -X POST http://localhost:3001/users/preferences \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "risk_tolerance":"High",
    "liquidity_need":"Low",
    "has_emergency_fund":true
  }'

# 5. Get Preferences
curl -X GET http://localhost:3001/users/preferences \
  -H "Authorization: Bearer YOUR_TOKEN"

│                      BUSINESS LOGIC                              │

UserService Methods:
  1. getProfile(userId)
     - Retrieves user without password
     - Throws 404 if user not found

  2. updateProfile(userId, updateDto)
     - Updates name and/or email
     - Checks email uniqueness
     - Returns updated user

  3. getPreferences(userId)
     - Retrieves user preferences
     - Returns null if not found

  4. createPreferences(userId, createDto)
     - Creates new preferences
     - Throws 409 if already exist
     - Links to user_id

  5. updatePreferences(userId, updateDto)
     - Updates existing preferences
     - Creates if not exist
     - Partial updates supported
